### 3. The 4 Questions

#### 1. What was expected to happen?
The browser was expected to block the mock script because `https://mock-unauthorized-tracker.com` was not whitelisted in the Vanguard CSP headers. This block was expected to generate a console error and shift the CISO SPC metric control light to Yellow.

#### 2. What actually occurred?
The browser blocked the script immediately, generated the exact CSP violation console error, and prevented the code execution. Playwright caught the error log, and we successfully recorded the Yellow status in the CAPA log.

#### 3. What went well and why?
*   **Rapid Defense:** The CSP successfully defended the page without needing any backend or runtime script intervention.
*   **Fast Recovery:** The rollback was completed and verified within 2 minutes of the initial alert.

#### 4. What can be improved and how?
*   *Observation:* Running a full Firebase Hosting deployment takes ~15–20 seconds for each toggle.
*   *Action:* We can configure local testing in the Firebase Emulator (`firebase emulators:start`) to verify CSP rule changes instantly without needing a full production deploy, using production deploy solely as a final confirmation step.
