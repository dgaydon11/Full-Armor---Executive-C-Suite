# TOKEN GOVERNANCE STANDING ORDER
**Document ID:** CQO-GOV-02  
**Owner:** Chief Quality Officer (Veritas) & Chief Financial Officer (CFO)

This standing order establishes the **Token Budgeting and Model Routing Architecture** for the Full Armor Studios ecosystem, designating the HQ Agent as the **Token Treasurer** responsible for enforcing the 3-try loop ceiling, model baseline locking, and the 3-tier traffic-light system.

---

## 1. 3-TIER REAL-TIME COST CONTROL SYSTEM
To ensure safe, automated code execution without manual overhead, operations are governed by three distinct states:

*   **🟢 GREEN LIGHT (Normal Operations | 0-80% of daily target):**
    *   *Action:* Code writing proceeds at full speed.
    *   *Reporting:* Silent logging of token metrics to local tracking storage. Zero chat noise.
*   **🟡 YELLOW LIGHT (Staff Review Gate | 80-100% of budget OR 2nd loop attempt):**
    *   *Action:* Non-blocking alert printed in chat. Coding continues while the Staff (CFO/CISO) reviews logs and adjusts task allocations.
    *   *Compliance:* **All Yellow Light events must be logged in [yellow_light_capa_log.md](file:///C:/Users/Don1/Desktop/Full-Armor---Executive-C-Suite/Quality%20-%20Chief-Quality_Officer/yellow_light_capa_log.md) along with the documented corrective action.**
*   **🔴 RED LIGHT (Circuit Breaker | 100%+ of budget OR 3rd loop attempt):**
    *   *Action:* **All programming dispatches stop immediately.** The system locks down and prompts the Human Operator for a manual override.

---

## 2. STRICT LOOP PREVENTION (3-TRY CEILING)
*   **Ceiling Rule:** No agent is permitted more than **3 attempts** to solve any single file edit, command error, or compile failure.
*   **Enforcement:** Upon a failed 3rd attempt, HQ must trigger the **Red Light Circuit Breaker**, freeze operations, and alert the operator.

---

## 3. MODEL BASELINE LOCKING
*   **Default Baseline:** All standard operations, coding tasks, and formatting are strictly locked to the **low-cost, high-efficiency Flash model tier**.
*   **Upgrade Limit:** Automated routing to higher Pro-tier models is disabled. Upgrades require a manual request to the operator with data justifying the complexity.
